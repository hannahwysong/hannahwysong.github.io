Team 3 Web Page

Team members: Dorian Earl-Front End, Hannah Wysong-Backend, Caleb Walton-Front/Backend, Sean Snook

Project Details

This project consists of designing and building a website that functions as a class registration portal for a user to use to register for classes with a major, minor, or combinaition of the two. This will consits of a website front end that will be used as an interface for the users to interact with and a MySql database as a backend to hold all of the class, major, and minor information.

Software Requirements

Components/Implementations
Front End
⦁ Web Server
⦁ Login page
Back End
⦁ SQL Database
⦁ Major and Minors
⦁ Contains fields: name, sub-categories, credit hours required total and or each sub-categories, classes for each sub-category
⦁ Course Data
⦁ Contains fields: Id, name, credit hours, description, pre-requisites, terms offered

Requirements
⦁ Select Two or more majors/minors
⦁ Generate a list of prerequisite courses
⦁ List of faculty
⦁ When the class is offered
⦁ Show the student if they can complete it by the time they graduate
⦁ Students: can select major and minors, create semester and year long schedules, request semester long co-ops for approval, view required courses, evaluate whether they can finish a degree in a specified number of years
⦁ Faculty: approve initial major and minors, can provide amendments to schedules, approve or deny co-ops, can change required courses, can provide guidance to student (message/get notified)
⦁ System Administrator: assign and remove student to faculty, add and remove student and faculty accounts, update usernames and passwords
⦁ All uses have username and password (can login)
⦁ All usernames must be 8-characters long, with only lowercase characters being allowed.
⦁ All passwords must be 12 characters long, with at least 1 uppercase, 1 lowercase, 1 numeric, and 1 symbol.
⦁ Single user interface (I assume one page)
⦁ Minimum two majors and two minors selected
⦁ 5 Students, 2 faculty and system administrators

⦁ Do not worry about "or" pre and co requisites
⦁ Simplified Scheduling where summer is not default but can be used
⦁ Form for co-op approval should include the semester and whether to extend the degree time period.
⦁ System should determin if possible and faculty can aprove or deny based on this
⦁ Basic error checking (ex. username too long or short)
⦁ Autogen schedules that can be changed after
⦁ Optional: editable course catalog by admin
⦁ Guidance is through a notification system
⦁ Below 11 or above 20 credits flags faculty for approval
⦁ Initial plans autogened need approval
